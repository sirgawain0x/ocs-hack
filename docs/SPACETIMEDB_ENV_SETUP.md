# SpacetimeDB Environment Variables Setup

## Required Configuration

Add these environment variables to your `.env.local` file in the root directory:

```bash
# ============================================================================
# SPACETIMEDB CONFIGURATION
# ============================================================================
# Client-side (browser) - must have NEXT_PUBLIC_ prefix
NEXT_PUBLIC_SPACETIME_HOST=https://maincloud.spacetimedb.com
NEXT_PUBLIC_SPACETIME_MODULE=beat-me

# Server-side (API routes) - no NEXT_PUBLIC_ prefix needed
SPACETIME_HOST=https://maincloud.spacetimedb.com
SPACETIME_MODULE=beat-me
# SPACETIME_TOKEN=  # Optional - only if you have a specific service token
```

## How This Works

### Module Name vs Database Identity

- **Module Name**: `beat-me` - This is the human-readable name you use to connect
- **Database Identity**: The long hex string is automatically assigned by SpacetimeDB
- **Connection**: The SDK uses the module name, not the database identity

### Anonymous Authentication

The app uses anonymous authentication which works as follows:

1. **First Connection**: SDK automatically generates a unique identity and token
2. **Token Storage**: The token is saved to `localStorage` as `spacetime_auth_token`
3. **Subsequent Connections**: The saved token is reused to maintain the same identity
4. **Persistence**: Your identity persists across browser sessions

### No Token Configuration Needed

You **DO NOT** need to set `SPACETIME_TOKEN` in your `.env.local` because:
- Anonymous auth is enabled
- Tokens are auto-generated by the SDK
- Tokens are managed in the browser's localStorage
- Each user gets their own persistent identity

## Quick Setup

1. **Create `.env.local`** in your project root if it doesn't exist
2. **Add the SpacetimeDB variables** shown above
3. **Keep your existing variables** (CDP_API_KEY, contract addresses, etc.)
4. **Restart your dev server** for changes to take effect

## Testing the Connection

After setting up your `.env.local`:

1. **Restart your dev server**:
   ```bash
   npm run dev
   ```

2. **Check browser console** for:
   ```
   ✅ Connected to SpacetimeDB with identity: <hex-string>
   ```

3. **Verify token persistence**:
   - Open DevTools → Application → Local Storage
   - Look for key: `spacetime_auth_token`
   - Value should be a JWT token string

4. **Test reconnection**:
   - Refresh the page
   - Should connect with the same identity
   - Console shows same identity hex string

## What Changed

### Before (WRONG ❌)
```typescript
// Manually building WebSocket URI with database identity
const wsUri = `wss://maincloud.spacetimedb.com/database/subscribe/${DATABASE_IDENTITY}`;
builder.withUri(wsUri).withModuleName("beat-me")
```

### After (CORRECT ✅)
```typescript
// SDK handles WebSocket URI construction
builder
  .withUri("https://maincloud.spacetimedb.com")  // Just the host
  .withModuleName("beat-me")  // Module name only
```

## Troubleshooting

### Issue: "SpacetimeDB initialization failed"

**Solution**: Ensure you have these exact variables in `.env.local`:
```bash
NEXT_PUBLIC_SPACETIME_HOST=https://maincloud.spacetimedb.com
NEXT_PUBLIC_SPACETIME_MODULE=beat-me
```

### Issue: Different identity on each connection

**Solution**: Check that token persistence is working:
1. Open DevTools → Console
2. Look for: `✅ Connected to SpacetimeDB with identity: ...`
3. Refresh page
4. Identity should be the same

If identity changes, check:
- localStorage is enabled in your browser
- No browser extensions blocking localStorage
- Check console for any localStorage errors

### Issue: Connection timeout

**Solution**:
1. Verify you're using `https://maincloud.spacetimedb.com` (not `ws://` or `wss://`)
2. Check that module name is exactly `beat-me`
3. Ensure you're not behind a firewall blocking WebSocket connections
4. Try clearing localStorage and reconnecting

## Security Notes

- `.env.local` is already in `.gitignore` - never commit it
- Auth tokens are stored in browser localStorage
- Each browser/device gets its own unique identity
- Tokens are managed automatically by the SDK
